## Executive Summary

ReelForge **already computes** creator readiness locally. The production signal (“25% readiness, 6 episodes missing assets”) is not a missing backend capability — it is a **visibility and aggregation problem**.

Readiness intelligence exists across ~15 modules but is:

- **Aggregate-first** (series %, not episode checklist)
- **Scattered** (Overview disclosures, Production tab widgets, Guide Me cards, Command Center)
- **Action-oriented elsewhere** (PRODUCT-02 solved attach; readiness UI still does not answer “what is complete per episode?”)

PRODUCT-04 recommends a **read-only Episode Readiness Board** built entirely from existing calculators — no new persistence, upload, or AI systems.

## 1. What Defines “Episode Ready”?

ReelForge uses **multiple readiness definitions** depending on subsystem. They overlap but are not unified in one UI.

### Canonical layers

| Layer | Definition | Source | “Ready” means |
|-------|------------|--------|----------------|
| **Playable** | `episodeIsPlayable()` | `seriesTypes.js` | `status ∈ {published, ready}` **and** `reelId` set |
| **Asset attached** | `resolveEpisodeAssetStatus()` | `episodeAssetStatus.js` | `reelId` set **and** reel found in `feedReels` |
| **Metadata complete** | `metadataCompleteForRecord()` | `productionHealth.js` | Title + description + genre + runtime |
| **Publishing** | Episode `status` | `seriesCatalog` | `published` vs `draft` / `ready` |
| **Pipeline gate** | `isPublishingBlocked()` | `episodePipeline.js` | Stage `READY` + `approvedBy` set |
| **Release scheduled** | Release Center | `releaseCenter.js` | Entry in `reelforge_release_schedule` with asset |

### Weighted series readiness (Guide Me / Command Center)

From `productionHealth.js` — **READINESS_WEIGHTS**:

| Pillar | Weight | Calculation |
|--------|--------|-------------|
| Metadata | 25% | Episodes with complete metadata / total |
| Assets | 35% | Episodes with reel in feed / total |
| Publishing | 25% | Episodes `Published` / total |
| Release schedule | 15% | Scheduled episodes with assets / scheduled |

**Weighted score** = sum of pillar % × weight. This drives the **25% readiness** seen in production Guide Me.

### Practical “creator-ready” definition (recommended for dashboard)

For PRODUCT-04 implementation, define **Episode Production Ready** as:

```text
✅ Reel attached     episode.reelId + reelInFeed
✅ Metadata complete metadataCompleteForRecord
✅ Thumbnail present feedReel.thumbnailUrl OR series poster fallback (optional tier)
✅ Publishing status status ∈ {ready, published} (optional tier)
⚠ Hero (series-level) reelforge_hero_manager_config — not per-episode
```

Hero is **site-wide**, not per-episode — show at series level, not duplicated on every row.

## 3. Gap Analysis: Actual Missing Features vs Missing Visibility

| Observation | Classification | Notes |
|-------------|----------------|-------|
| 25% readiness, 6 missing assets | **Visibility gap** | `getMissingAssetQueue()` already returns queue |
| No per-episode checklist UI | **UI gap** | `buildEpisodeOperationRows()` has row-level data |
| Guide Me says “attach E01” but creator hunts UI | **Partially solved** | PRODUCT-02 attach panel; readiness board would close loop |
| Hero “missing” on episode row | **Model gap** | Hero is series/site level — wrong granularity if per-episode |
| Thumbnail not in `EpisodeAssetRecord` | **Schema gap (minor)** | `actionEngine` has `missing-thumbnail` type; not on asset record |
| Readiness buried in `<details>` on Overview | **UX gap** | Production tab shows aggregate meters only |
| Workflow tasks vs readiness pillars disconnected | **Presentation gap** | Same underlying rows, different surfaces |
| Studio DB vs catalog ID bridge | **Solved** | PRODUCT-02 orchestration |
| Upload / ingest broken | **Not a gap** | BG phase closed; frozen foundation |

**Conclusion:** PRODUCT-04 should **expose existing state**, not invent new readiness logic or storage.

## 4. Signals Calculable Locally (No New Backend)

All of the following can be derived at runtime from `feedReels + seriesCatalog + localStorage`:

| Signal | Calculation |
|--------|-------------|
| Reel attached | `row.reelId && row.reelInFeed` |
| Missing asset | `row.status === 'Missing Asset'` |
| Metadata gaps | `!row.metadataComplete` → sub-checks via `metadataGaps()` in actionEngine |
| Publishing state | `row.publishingStatus` / `episode.status` |
| Release scheduled | Join `releaseCenter` schedule by `episodeId` |
| Pipeline review | `getPipelineReviewStatus(episodeId)` |
| Next recommended action | Top item from `buildStudioActionPlan()` for that `episodeId` |
| Readiness impact | `StudioRecommendation.impact` from actionEngine |
| Projected readiness | `syncWorkflowTasks().projected` |
| Hero site status | `loadHeroManagerConfig()` — series-level banner |
| Vault reel available | Count non-hero `personalVideos` |

No AI, no new API, no upload changes required.

## 9. Answers to Mission Questions

### 1. What defines “episode ready”?

**System truth:** Playable = published/ready status + reel in feed.  
**Creator truth (recommended):** Reel attached + metadata complete + (optional) thumbnail + publishing status.  
**Series score:** Weighted pillars in `productionHealth.js`.

### 2. Which data sources already contain readiness signals?

`seriesCatalog`, `reelforge_series_metadata`, `feedReels`, vault/hero localStorage, release schedule, pipeline store, workflow task generator, actionEngine, guideMeEngine, commandCenter.

### 3. Which gaps are actual missing features?

- Per-episode readiness **dashboard UI** (primary)
- Thumbnail flag on asset record (minor — derivable from feed reel)
- Hero per-episode (wrong model — keep series-level)

Not missing: attach workflow (PRODUCT-02), attach automation (PRODUCT-03), readiness **math**.

### 4. Which signals can be calculated locally?

All pillar scores, per-episode asset/metadata/publishing status, missing-asset queue, workflow projections, hero site status, recommended next action.

### 5. What is the smallest useful creator dashboard?

**Episode Readiness Board** on Production tab: one row per episode, checkmarks for reel/metadata/thumbnail, single CTA linking to existing PRODUCT-02 attach or metadata editor. Reuse `buildEpisodeOperationRows` — zero new identity systems.
